# Kittygram Docker 🐱

## 📝 Полное описание проекта, целей и его функций

**Kittygram** — это социальная сеть для любителей кошек. Проект разработан как fullstack-приложение, позволяющее пользователям делиться фотографиями своих питомцев, вести учет их достижений и возраста.

### Основные цели проекта:
* **Контейнеризация приложения:** перевод локального монолита на микросервисную Docker-архитектуру для изоляции процессов бэкенда, фронтенда и базы данных.
* **Автоматизация деплоя и runtime-процессов:** минимизация ручных операций (инициализация томов, запуск и связывание сервисов через Docker Compose).
* **Обеспечение безопасности:** защита персональных данных пользователей, скрытие конфиденциальных ключей в переменных окружения и настройка безопасного реверс-прокси сервера.

### Функциональные возможности:
* **Профиль питомца:** добавление котиков с указанием имени, возраста, цвета и уникальных достижений.
* **Галерея фотографий:** загрузка и хранение изображений питомцев (реализовано через защищенные Docker-volumes для медиафайлов).
* **Архитектурная изоляция:** веб-сервер принимает внешние запросы на порту `9000`, самостоятельно отдает статические ресурсы фронтенда и перенаправляет API-запросы на бэкенд-сервер.

## 🛠 Стек применённых технологий

### Backend & API:
* **Python 3.10** — основной язык разработки бэкенда.
* **Django & Django REST Framework (DRF)** — веб-фреймворк для построения RESTful API.
* **Gunicorn** — WSGI HTTP-сервер для запуска Django в продакшн-окружении.

### Frontend:
* **Node.js / React** — библиотека для построения динамического пользовательского интерфейса (SPA).

### База данных & Инфраструктура:
* **PostgreSQL 13** — реляционная база данных для надежного хранения информации.
* **Nginx** — веб-сервер и реверс-прокси для маршрутизации запросов и отдачи статики фронтенда.
* **Docker & Docker Compose** — оркестрация, контейнеризация и управление зависимостями проекта.
* **GitHub Actions** — автоматизация тестирования и линтинга кода (CI/CD пайплайн).

## 🚀 Инструкция по развёртыванию проекта

### 1. Подготовка окружения на сервере (виртуалке)
Перед запуском контейнеров убедитесь, что порт `9000` свободен на хост-машине. Если на этом порту работает системный Nginx или другой сервис, его необходимо остановить или перенастроить:
```bash
# Пример остановки системного nginx, если он занимает нужный порт
sudo systemctl stop nginx
sudo systemctl disable nginx
```

### 2. Клонирование репозитория
Склонируйте проект с GitHub и перейдите в его корневую директорию:
```bash
git clone https://github.com/xlllaMaH/kittygram-docker.git
cd kittygram-docker
```

### 3. Заполнение файла `.env` (Переменные окружения)
В корневой папке проекта (на одном уровне с файлом `docker-compose.yml`) создайте файл **`.env`**. Он необходим для безопасного хранения секретов.

```bash
nano .env
```

Скопируйте в него следующие параметры и обязательно замените значения паролей и ключей на свои собственные:

```env
# --- Настройки Django ---
# Секретный ключ проекта (не должен попадать в публичный код)
SECRET_KEY=django-insecure-ваш_уникальный_длинный_секретный_ключ
# Разрешенные IP-адреса вашей виртуалки и localhost через запятую без пробелов
ALLOWED_HOSTS=172.18.210.239,localhost,(Ваш IP адрес ВМ)
# Часовой пояс проекта
TIME_ZONE=(Ваш часовой пояс)

# --- Настройки базы данных PostgreSQL ---
# Имя пользователя БД
POSTGRES_USER=kittygram_user
# Пароль для доступа к БД (сделайте его максимально сложным)
POSTGRES_PASSWORD=kittygram_secure_password
# Название базы данных
POSTGRES_DB=kittygram
# Имя сервиса БД из docker-compose.yml
DB_HOST=db
# Стандартный порт PostgreSQL
DB_PORT=5432
```

### 4. Запуск Docker Compose
Запустите сборку образов и старт всех контейнеров в фоновом режиме одной командой:
```bash
sudo docker compose up -d
```

Docker Compose автоматически инициализирует тома (static, media, pg_data), поднимет базу данных, запустит сборку фронтенда и настроит конфигурацию Nginx для проксирования запросов.
## В данной конфигурации автоматический сбор статики для бэкенда (collectstatic) не выполняется при запуске. Если вам потребуется обновить статические файлы админ-панели Django после изменений, выполните команду вручную:
```bash
sudo docker compose exec backend python manage.py collectstatic
sudo docker compose exec backend cp -r /app/collected_static/. /backend_static/static/
```

### 5. Создание суперпользователя (Администратора)
Чтобы войти в панель управления Django, создайте аккаунт администратора внутри запущенного контейнера:
```bash
sudo docker compose exec backend python manage.py createsuperuser
```
